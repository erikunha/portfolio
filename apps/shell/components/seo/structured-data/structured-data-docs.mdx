## Schema Types

### Person Schema

**When to use**: Personal portfolio, about pages, author pages

**Includes**:

- Basic info (name, job title, description)
- Social media profiles (`sameAs`)
- Skills (`knowsAbout`)
- Website URL

**Benefits**:

- Rich results in search (Knowledge Panel)
- Social media link validation
- Professional credibility

### WebSite Schema

**When to use**: Homepage, main landing page

**Includes**:

- Site name and URL
- Author information (nested Person)

**Benefits**:

- Site name in search results
- Sitelinks search box eligibility
- Better search understanding

## Best Practices

### 1. **One Primary Schema Per Page**

<Source
  language="tsx"
  code={`

### 2. **Match Content to Schema**

Only use Person schema on pages about you. Use appropriate schemas for other content types.

Update social links and skills as they change:

  language="tsx"

Always test with Google's Rich Results Test after changes.
