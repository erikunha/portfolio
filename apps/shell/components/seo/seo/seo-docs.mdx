# SEO Component

The SEO component is a wrapper around `next-seo` that provides consistent metadata management across the portfolio.

### Blog Post with Custom Metadata

<Source
  language="tsx"
  code={`
import { SEO } from '@/components/seo';

export default function BlogPost({ post }) {
  return (
    <>
      <SEO
        title={\`\${post.title} - Erik Henrique\`}
        description={post.excerpt}
        url={\`https://erikunha.dev/blog/\${post.slug}\`}
        image={post.coverImage}
        // Additional next-seo props
        openGraph={{
          type: 'article',
          article: {
            publishedTime: post.publishedAt,
            authors: ['Erik Henrique Alves Cunha'],
          },
        }}
      />
      {/* Your page content */}
    </>
  );
}
`}
/>

## Props API

| Prop          | Type           | Default                   | Description      |
| ------------- | -------------- | ------------------------- | ---------------- |
| `title`       | `string`       | `DEFAULT_SEO.title`       | Page title       |
| `description` | `string`       | `DEFAULT_SEO.description` | Meta description |
| `url`         | `string`       | `DEFAULT_SEO.url`         | Canonical URL    |
| `image`       | `string`       | `DEFAULT_SEO.image`       | OpenGraph image  |
| `...rest`     | `NextSeoProps` | -                         | Any NextSeo prop |

## Best Practices

### 1. **Always Use Canonical URLs**

<Source
  language="tsx"
  code={`
<SEO
  url="https://erikunha.dev/about"  // ‚úÖ Absolute URL
  // NOT: url="/about"              // ‚ùå Relative URL
/>
`}
/>

### 2. **Optimize Descriptions**

- Keep between 150-160 characters
- Include primary keywords naturally
- Make it compelling for click-through

### 3. **Use High-Quality Images**

- Minimum 1200x630px for OpenGraph
- Use descriptive alt text
- Optimize file size (< 1MB)

### 4. **Update Per Page**

Don't rely on defaults for every page - customize metadata to match page content.

## Testing
