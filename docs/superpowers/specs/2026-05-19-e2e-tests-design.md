# E2E Tests Spec — Design

> Validated through `superpowers:brainstorming` 2026-05-19. Closes a documented-but-unbuilt coverage gap: CLAUDE.md lines 16 + 77 claim "Playwright E2E (contact + ask paths)" exist; they never did. Only `tests/e2e/observability-smoke.spec.ts` (shipped in PR #11) had ever existed in repo history before this spec.

## 1. Goal

Build the user-journey end-to-end coverage CLAUDE.md describes, plus visual regression and cross-browser breadth, on top of the existing source-grep unit suite and the `tests/a11y/axe.spec.ts` accessibility scan. The unit tests already cover route handler behavior at the source level; this spec covers **what an actual visitor experiences in a browser** — and what regresses when CSS/markup/handler changes interact.

## 2. Non-goals

- Replacing source-grep unit tests. The grep pattern stays the dominant unit shape.
- Testing third-party SDKs (Anthropic, Resend, Upstash). Backend is mocked at the route boundary.
- Performance budgets. Lighthouse CI is the perf gate; e2e tests assert behavior + visual, not metrics.
- Authentication / multi-user flows. The portfolio is single-author single-anonymous-visitor.

## 3. Background context

### What exists today

| Layer | Surface | Status |
|---|---|---|
| Source-grep unit tests | `__tests__/*.test.ts` (~125 specs) | Active gate; covers route handler shape, content schemas, CSS drift |
| A11y scan | `tests/a11y/axe.spec.ts` (chromium, 2 tests) | Active gate; covers homepage + contact form error state |
| E2E observability smoke | `tests/e2e/observability-smoke.spec.ts` (chromium, 4 tests) | Active gate; covers `/api/log` + `/api/log/forget` HTTP shape |
| **User-journey E2E** | — | **MISSING** (this spec) |
| **Visual regression** | — | **MISSING** (this spec) |
| **Cross-browser** | — | **MISSING** (this spec) |

### CI shape today (post PR #13)

`build-and-gate` (Lighthouse + axe-core + unit + build + bundle) and `e2e` (observability smoke; gracefully skips when `tests/e2e/*.spec.ts` is empty). One `e2e` job runs chromium only.

### Constraints

- Project rejects external SaaS where local primitives suffice (per `DECISIONS.md` 2026-05-13 architecture pass). Visual regression service like Percy is out.
- Pure-black background + lime-green accents + CRT scan beam + matrix loop create extreme rendering variance per frame; any naive screenshot diff will flake on every run.
- CI budget: existing build-and-gate is ~3 min, e2e is ~1 min. Adding a 4-project matrix can plausibly add 5-8 min — acceptable but worth measuring.

## 4. Validated design decisions (from brainstorming)

| Question | Decision |
|---|---|
| Scope | Full coverage incl. visual regression + cross-browser |
| Backend exercise | Mock at the route level via Playwright `page.route()` |
| Visual regression management | Threshold-based pixel-diff (`maxDiffPixelRatio: 0.01`) with anti-flake mask on CRT scan beam, matrix loop, typing cursor, timestamps, animated boot transcript |
| Browser matrix | Chromium + WebKit, desktop (1280×720) + mobile (375×667 / 390×844) → 4 Playwright projects |
| Delivery shape | Anchor PR (~6 tests, chromium-desktop only) → expansion PRs (rest of matrix + remaining tests) |
| CI integration | Split `e2e-full` job (4-project matrix, `needs: build-and-gate`) from existing `e2e` smoke job (which keeps the observability smoke) |

## 5. Architecture

### File layout

```

    axe.spec.ts                          (existing — unchanged)

```

### Snapshot wrapper (sketch)

```ts
// tests/e2e/_helpers/snapshot.ts
import { expect, type Page } from '@playwright/test';
import { volatileMasks } from './mask-volatile';

export async function snapshot(page: Page, name: string) {
  await expect(page).toHaveScreenshot(name, {
    mask: volatileMasks(page),
    maxDiffPixelRatio: 0.01,
    animations: 'disabled',
  });
}
```

## 6. Test surface

22 tests × 4 projects (where applicable) = ~70-80 total runs.

### Contact form (5 tests)

| # | Behavior | Mock state |
|---|---|---|
| 1 | Happy path: fill all fields → submit → success state | `contact: 'happy'` |
| 2 | Validation: blank submit shows inline errors + axe-checks the error state | `contact: 'validation-error'` |
| 3 | Rate-limited: 429 response → user-visible "try again" + retry-after | `contact: 'rate-limit'` |
| 4 | Server-error: 500 → graceful error UI, no crash | `contact: 'server-error'` |
| 5 | Honeypot trip: hidden field filled → submit succeeds silently | `contact: 'honeypot'` |

### Ask flow (8 tests)

| # | Behavior | Mock state |
|---|---|---|
| 1 | Happy path: typed question → streamed answer renders progressively | `ask: 'happy'` |
| 2 | Kill switch on: 503 → "temporarily unavailable" | `ask: 'kill-switch'` |
| 3 | Rate-limit: 429 → "try again in an hour" | `ask: 'rate-limit'` |
| 4 | Budget exhausted: 503 budget message | `ask: 'budget-exhausted'` |
| 5 | Stream error: emit `STREAM_ERR_SENTINEL + msg` → error rendering | `ask: 'stream-error'` |
| 6 | Pre-stream timeout: backend rejects before SSE starts (Spec 1 fix) | `ask: 'pre-stream-timeout'` |
| 7 | `X-Request-Id` header surfaced: assert present + non-empty | `ask: 'happy'` |
| 8 | Privacy notice visible + email link works | (no mock needed) |

## 7. Data flow

1. Playwright test starts → `installMockBackend(page, state)` registers `page.route()` interceptors.
2. `page.goto('/')` loads the Next.js dev/preview server (real RSC + client islands).
3. User action triggers `fetch('/api/...')` from the client.
4. Playwright intercept catches the request → returns canned response per state.
5. UI renders the response; test asserts visible behavior + optional snapshot.

The mock layer is at the network boundary. The full RSC → client island → fetch → response → UI render path is exercised end-to-end except for the route handler internals (which the source-grep tests cover).

## 8. Error handling

- **Missing mock fixture:** helpers `throw` (no silent pass). A test that hits an un-mocked endpoint surfaces immediately.
- **Test isolation:** `test.beforeEach` resets storage / cookies / route mocks.
- **Retries:** 2 in CI (existing), 0 locally. Trace on first retry (existing).
- **Snapshot baseline:** `__snapshots__/` is gitignored when first added (anchor PR) → committed once stable, then enforced.

## 9. CI integration

### Workflow shape

Existing `build-and-gate` and `e2e` (smoke) jobs stay as-is. Add a new `e2e-full` job:

```yaml
e2e-full:
  runs-on: ubuntu-latest
  timeout-minutes: 20
  needs: build-and-gate
  strategy:
    fail-fast: false
    matrix:
      project: [chromium, chromium-mobile, webkit-desktop, webkit-mobile]
  steps:
    - uses: actions/checkout@v4
    - uses: pnpm/action-setup@v4
    - uses: actions/setup-node@v4
      with: { node-version: 22, cache: pnpm }
    - run: pnpm install --frozen-lockfile
    - run: pnpm playwright install --with-deps ${{ contains(matrix.project, 'webkit') && 'webkit' || 'chromium' }}
    - run: pnpm build
      env:
        ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY_BUILD }}
        UPSTASH_REDIS_REST_URL: ${{ secrets.UPSTASH_REDIS_REST_URL_BUILD }}
        UPSTASH_REDIS_REST_TOKEN: ${{ secrets.UPSTASH_REDIS_REST_TOKEN_BUILD }}
        RESEND_API_KEY: ${{ secrets.RESEND_API_KEY_BUILD }}
        DEPLOY_SALT: ci-build-salt
    - name: Check if e2e-full tests exist
      id: check-full
      run: |
        if ls tests/e2e/{contact,ask,visual,cross-cutting}.spec.ts 2>/dev/null | grep -q .; then
          echo "exists=true" >> "$GITHUB_OUTPUT"
        else
          echo "exists=false" >> "$GITHUB_OUTPUT"
        fi
    - if: steps.check-full.outputs.exists == 'true'
      run: pnpm start &
    - if: steps.check-full.outputs.exists == 'true'
      run: npx wait-on http://localhost:3000 --timeout 30000
    - if: steps.check-full.outputs.exists == 'true'
      run: pnpm playwright test --project=${{ matrix.project }} tests/e2e/{contact,ask,visual,cross-cutting}.spec.ts
    - uses: actions/upload-artifact@v4
      if: failure()
      with:
        name: playwright-report-${{ matrix.project }}
        path: playwright-report/
```

Anchor PR makes `e2e-full` non-required (informational only). Once anchor stabilizes and expansion PRs land, promote to required.

### CI budget impact

- Per-project run: ~2-3 min (build + install + tests + matrix coordination)
- 4 projects in parallel: ~3 min wall-clock (assuming GH Actions has enough concurrency)
- Worst-case CI cost: existing ~4 min + new ~3 min = ~7 min total

## 12. Success criteria

1. `tests/e2e/contact.spec.ts` + `tests/e2e/ask.spec.ts` + `tests/e2e/visual.spec.ts` + `tests/e2e/cross-cutting.spec.ts` exist with the specified tests

4. Backend mocks cover every documented `MockState` variant
5. Volatile-region masks cover CRT, matrix, cursor, timestamps, boot transcript
6. Snapshot baselines committed and stable across 5 consecutive CI runs without manual update
7. CLAUDE.md updated to remove the false "Playwright E2E (contact + ask paths)" claim
8. New ADR in `DECISIONS.md` documenting the e2e architecture decisions (mock-backend chosen over real APIs, pixel-diff with masks chosen over Percy/Chromatic, 4-project matrix chosen over Firefox+full or chromium-only)

## 13. Risks + mitigations

| Risk | Mitigation |
|---|---|
| Snapshot flakiness from font rendering across CI runners | `maxDiffPixelRatio: 0.01` tolerance + `animations: 'disabled'` + JetBrains Mono is self-hosted (deterministic) |
| WebKit-specific CSS quirks (esp. CRT effects with `backdrop-filter`) | Separate baseline per project; webkit can have its own snapshot |
| CI minute spend doubles | Anchor PR is non-required; CI cost measured before promotion to required |
| Mock drift from real route handler shape | Source-grep tests already enforce route shape; mock fixtures cite the assertion that generates them |
| Visual regression baselines get stale during active design work | Phase 4 (promote-to-required) is gated on 5 consecutive stable runs — don't promote during a design churn window |

## 14. Reversibility

Entire spec is reversible by:
- Deleting `tests/e2e/{contact,ask,visual,cross-cutting}.spec.ts`
- Deleting `tests/e2e/_helpers/`
- Deleting `tests/e2e/__snapshots__/`
- Reverting `playwright.config.ts` to single-project
- Removing the `e2e-full` workflow job

The observability smoke (`tests/e2e/observability-smoke.spec.ts`) and the a11y scan (`tests/a11y/axe.spec.ts`) stay either way.
