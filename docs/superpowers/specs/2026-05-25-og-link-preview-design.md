# OG Link Preview Fix — Design Spec

**Date:** 2026-05-25
**Status:** Approved
**Scope:** Fix link preview on WhatsApp, LinkedIn, Slack, Discord, Twitter/X, and any platform that reads Open Graph tags.

---

## Problem

`app/opengraph-image.tsx` runs on the Edge runtime and loads the JetBrains Mono font via `new URL('../public/fonts/...', import.meta.url)`. This pattern is unreliable in Edge runtime production environments (filesystem access differs from local dev). The endpoint has never been verified live. Additionally, the `openGraph` and `twitter` metadata blocks in `layout.tsx` carry no explicit `images` field — they rely on Next.js auto-detection of the Edge function, which silently fails if the function errors. The result: all link-preview crawlers see no `og:image` tag.

---

## Approach: Static PNG + explicit metadata (Approach A)

Replace the dynamic Edge-generated image with a committed static PNG. Single URL portfolio — no per-page dynamic previews are needed.

---

## File changes

| File | Action |
|---|---|
| `app/opengraph-image.tsx` | Delete |
| `public/og.png` | Add — 1200×630, generated by Playwright screenshot of the existing Edge function design (dev mode) |
| `scripts/generate-og-image.ts` | Add — one-time generator; kept for future redesigns |
| `app/layout.tsx` | Add explicit `images` to `openGraph` and `twitter` blocks |
| `__tests__/og-metadata.test.ts` | Add — regression guard on `metadata.openGraph.images` |

---

## Image generation

The script (`scripts/generate-og-image.ts`):
1. Starts the Next.js dev server (`pnpm dev`)
2. Waits for it to be ready
3. Navigates Playwright to `http://localhost:3000/opengraph-image` at exactly 1200×630 viewport
4. Takes a full-page screenshot, saves to `public/og.png`
5. Exits

This reuses the existing JSX design in `opengraph-image.tsx` without re-implementing it. After committing the PNG, the Edge function is deleted.

## Metadata change (`app/layout.tsx`)

```ts
openGraph: {
  type: 'website',
  locale: 'en_US',
  url: 'https://erikunha.dev',
  title: 'Erik Cunha — Staff Full-Stack Engineer · Applied AI',
  description: 'Staff Full-Stack Engineer · Applied AI · 8+ yrs · LLM · RAG · Angular · React · Next.js · Node.js',
  siteName: 'erikunha.dev',
  images: [
    {
      url: '/og.png',
      width: 1200,
      height: 630,
      alt: 'Erik Cunha — Staff Full-Stack Engineer · Applied AI',
    },
  ],
},
twitter: {
  card: 'summary_large_image',
  title: 'Erik Cunha — Staff Full-Stack Engineer · Applied AI',
  description: 'Staff Full-Stack Engineer · Applied AI · Angular · React · Next.js · Node.js · TypeScript',
  images: ['/og.png'],
},
```

`metadataBase` is already `https://erikunha.dev` — Next.js resolves `/og.png` to the required absolute URL.

---

## Regression test (`__tests__/og-metadata.test.ts`)

Assert that `metadata.openGraph.images` is a non-empty array and that the first entry has `url`, `width`, and `height`. No snapshot — just shape and presence. Prevents silent regression from a future metadata refactor stripping the field.

## Crawler compatibility

| Platform | Requirement | Status after fix |
|---|---|---|
| WhatsApp | `og:image` absolute URL, ≥200×200 | 1200×630 passes |
| LinkedIn | `og:image` ≥1200×627 | Exact match |
| Slack | `og:image` + `og:title` + `og:description` | All present |
| Discord | Same as Slack | All present |
| Twitter/X | `twitter:card=summary_large_image` + `twitter:image` | Explicit field added |

**WhatsApp cache note:** Previews are cached up to 72 hours per URL. New shares show the updated preview immediately; existing cached previews expire naturally.

## robots.txt — verified, no change needed

`public/robots.txt` already exists with `User-agent: *` and `Allow: /`. All OG crawlers are covered unconditionally. Sitemap is referenced at `https://erikunha.dev/sitemap.xml` and `app/sitemap.ts` exists.

## Verification (post-deploy)

Run the following OG debugger tools against `https://erikunha.dev`:
- WhatsApp: share the URL on WhatsApp directly (no official debugger)
- LinkedIn: Post Inspector — `https://www.linkedin.com/post-inspector/`
- Slack: paste URL in a DM to yourself
- Twitter/X: Card Validator — `https://cards-dev.twitter.com/validator`
- General: `https://opengraph.dev` or `https://metatags.io`

All should show the 1200×630 black/green image with title and description.

## Out of scope

- OG video / OG audio tags
