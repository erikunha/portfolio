## 2026-05-21 — Agent surfaces: MCP `ask_erik` rate-limit scope (Phase 1, Task 1.5)

- **2026-05-21** · **MCP `ask_erik` shares one global rate-limit bucket — known, accepted limitation.** `runAsk` in `lib/agent/mcp-tools.ts` builds a synthetic same-origin `Request` to reuse the real `/api/ask` POST handler but sets no `x-forwarded-for` / `x-real-ip` header, so `getClientIp` (`lib/rate-limit.ts`) returns the literal `'unknown'`. ## 2026-05-21 — `/api/ask` migrated to the Vercel AI Gateway (Phase 1, Task 1.2)

- **2026-05-21** · **Spike outcome — PATH A: the Vercel AI Gateway exposes the cache-token breakdown, so the full migration is viable.** Task 1.1 (the P1.1 spike) confirmed against Vercel's AI Gateway docs + the AI SDK Anthropic-provider docs that `streamText` routed through the Gateway returns `cacheCreationInputTokens` and `cacheReadInputTokens` under `result.providerMetadata.anthropic`, and `inputTokens` / `outputTokens` under `result.usage`. Usage now arrives via the resolved end-of-stream promises (`result.usage` / `result.providerMetadata`) rather than a `message_start` SSE event. The PATH B fallback (keep the direct `@anthropic-ai/sdk` purely for the usage path) was therefore not needed — the cache-hit metric survives the migration intact. _Residual verification: (1) the doc-level confirmation is authoritative for the cache-token field names, but a live runtime smoke-test against a deployed preview (one warm + one cold request, asserting `cacheCreationInputTokens > 0` on the cold call) is the final check; it needs the deploy-env `AI_GATEWAY_API_KEY` and is listed as a deploy-env action. (2) The zero-refund error path depends on a load-bearing AI SDK assumption: that on a suppressed failed / init-failed stream `result.usage` RESOLVES to zeroed counts rather than REJECTING. The route's `usagePromise` `.catch` already degrades a rejection to `undefined` (treated as zero), and `settleAndPersist` gates the refund on real `usage` alone — so a rejection still settles fail-closed — but the resolves-to-zeros behavior is what makes the error-path refund a clean no-op; the same preview smoke-test should also exercise one forced-error request and confirm the budget settles without a phantom high-water mark._
- **2026-05-21** · **`/api/ask` migrated from the direct `@anthropic-ai/sdk` to the Vercel AI Gateway via the AI SDK v6 (`ai` package) `streamText`.** The Anthropic call and usage extraction are the only things that changed; every other behavior is preserved verbatim — the kill switch, per-IP rate limit, JSON-parse + 500-char clamp, the `INJECTION_RE` prompt-injection guard, the identical-question gate, the budget reserve/settle, the per-request unguessable `<q SENTINEL>` wrapper, the `STREAM_ERR_SENTINEL` error path, the mid-stream watchdog, the persisted `persistAskInteraction` log, and all response headers. The model is addressed by the plain Gateway string `anthropic/claude-haiku-4-5` (Vercel-preferred form — no `@ai-sdk/anthropic` instance wired directly); the ephemeral prompt cache is preserved by sending the system prompt as a `system`-role message carrying `providerOptions: { anthropic: { cacheControl: { type: 'ephemeral' } } }`. Rationale: the Gateway is one billing/observability/rate-limit seam in front of the model, the foundation Phase 1 builds the AI eval suite and metrics panel on. Notable mechanical change: `streamText` is synchronous and surfaces *both* stream-init and mid-stream failures through the `textStream` async iterator, so the route's former pre-stream `try/catch` around `messages.create()` collapses into the single mid-stream consumer catch — same observable contract (HTTP 200 + `STREAM_ERR_SENTINEL` body). Usage settlement (`settleBudget` + `persistAskInteraction`) now runs after the stream closes, awaiting `result.usage` / `result.providerMetadata`, because those resolve at stream end rather than on a `message_start` event. `cacheControl`, the `RESERVED_INPUT_TOKENS` budget sizing, and the cache-hit-rate metric (STANDARDS.md Ch.7) are unchanged. New deploy-env requirement: `AI_GATEWAY_API_KEY` (on Vercel, the OIDC token is used automatically). The CLAUDE.md stack line still naming the direct SDK is now superseded by this entry. `@anthropic-ai/sdk` is retained as a dependency — `lib/ask/system-prompt.ts` still uses its `TextBlockParam` type for the `SYSTEM` export, and it is the documented PATH B fallback. _Reversible: revert this commit — the direct-SDK `messages.create()` call returns; `pnpm remove ai` if desired. The cache-metric is pinned by `__tests__/ask-cache-metric.test.ts`, which is implementation-agnostic and passes on either path._

## 2026-05-19 — PR #29 review resolutions (3 Principal-grade fixes)

- **2026-05-19** · **`lib/inline-css.ts` minifier no longer strips whitespace around `+`** (review finding 1 on PR #29). The previous strip class `[{}:;,>+~]` was correct for CSS combinators but corrupted `calc(80px + env(safe-area-inset-bottom, 0px))` → `calc(80px+env(...))` — invalid CSS that browsers would silently fail to apply (the `+env(...)` parses as a unitless sign-positive literal instead of arithmetic). The codebase has 4+ calc() expressions in `_chrome.css` and `_layout.css` that were being broken on every inlined page render. Fix: remove `+` from the strip class — `[{}:;,>~]`. `-` was already absent and stays absent for the same reason. Trade-off: we lose ~5 bytes of minification savings on adjacent-sibling combinators (`.a + .b` no longer collapses) in exchange for correctness on every calc(). - **2026-05-19** · **`RESERVED_INPUT_TOKENS` raised from 1000 → 2200** (review finding 2 on PR #29). The original 1000 was sized for the pre-PR-4 ~750-token SYSTEM. ## 2026-05-19 — CSP nonce revert (PR 4 of audit roadmap)
